## Getting Started

PromptForge is an AI-powered platform designed to help you create, analyze, and enhance prompts for Large Language Models (LLMs) like ChatGPT, Claude, and Gemini.

### What Can PromptForge Do?

- **Analyze** your prompts for quality, clarity, and effectiveness
- **Enhance** prompts with AI-powered suggestions
- **Score** prompts across multiple dimensions
- **Create** reusable templates for common tasks
- **Track** prompt versions and improvements

### Who Is This For?

- Content creators and writers
- Developers building AI applications
- Product managers designing AI features
- Educators and researchers
- Anyone working with AI language models

## Analyzing Prompts

Prompt analysis is PromptForge's core feature. Get detailed insights into your prompt quality.

### Step 1: Create or Select a Prompt

**Option A: Create New Prompt**
1. Click **"New Prompt"** in the navigation
2. Fill in the prompt form:
   - **Title**: Descriptive title (e.g., "Blog Post Generator")
   - **Content**: Your actual prompt text
   - **Target LLM**: Select the AI model (ChatGPT, Claude, Gemini, etc.)
   - **Category**: Choose a category (Content, Code, Analysis, etc.)
   - **Tags**: Add relevant tags (optional)
3. Click **"Save Prompt"**

**Option B: Select Existing Prompt**
1. Navigate to **"Prompts"**
2. Click on any prompt to view details
3. Click **"Analyze"** button

### Step 2: Run Analysis

1. On the prompt detail page, click **"Analyze Prompt"**
2. Wait for AI analysis (typically 3-5 seconds)
3. View comprehensive results:

### Understanding Your Analysis Results

#### Overall Quality Score (0-100)

Your prompt receives an overall quality score based on four key metrics:

**üéØ Clarity Score (0-100)**
- How clear and understandable your prompt is
- Checks for ambiguous language
- Evaluates sentence structure

**üîç Specificity Score (0-100)**
- How specific and detailed your instructions are
- Checks for concrete examples
- Evaluates detail level

**üìê Structure Score (0-100)**
- How well-organized your prompt is
- Checks for logical flow
- Evaluates formatting

**üí° Best Practices Score (0-100)**
- How well your prompt follows LLM best practices
- Checks for context, constraints, examples
- Evaluates output specifications

#### Score Interpretation

| Score | Rating | Meaning |

| 75-89 | Good | Effective prompt, minor improvements possible |

| 0-39 | Very Poor | Major problems, complete rewrite recommended |

## Enhancing Prompts

After analyzing your prompt, PromptForge can automatically enhance it using AI.

### How to Enhance a Prompt

1. Analyze your prompt first (see above)
2. Review the analysis results
3. Click **"Enhance Prompt"** button
4. Wait for AI enhancement (5-10 seconds)
5. Review the enhanced version

### Enhancement Options

**üîÑ Standard Enhancement**
- Improves clarity and structure
- Adds missing context
- Follows best practices
- Single enhanced version

**üé® Multiple Variations**
- Generates 3 different enhanced versions
- Each focuses on different aspects:
  - Version 1: Clarity-focused
  - Version 2: Specificity-focused
  - Version 3: Comprehensive enhancement
- Compare and choose the best one

### Comparing Original vs Enhanced

The enhancement view shows:

**Side-by-Side Comparison**
- Original prompt on the left

**Quality Improvement Metrics**

- Estimated effectiveness gain

**What Changed**
- Added context and background
- Clarified instructions
- Specified output format
- Added constraints and examples
- Improved structure

### Using Enhanced Prompts

## Using Templates

Templates help you quickly create prompts for common tasks.

### Browsing Templates

1. Navigate to **"Templates"** in the menu
2. Browse available templates:
   - **Your Templates**: Templates you created
   - **Public Templates**: Community-shared templates
3. Use filters:
   - Category (Content, Code, Analysis, etc.)
   - Tags
   - Author
   - Popularity

### Using a Template

1. Find a template you like
2. Click on the template to view details
3. Review template structure and placeholders
4. Click **"Use Template"**
5. Fill in the placeholders:
   - `{topic}` ‚Üí Your specific topic
   - `{audience}` ‚Üí Target audience
   - `{format}` ‚Üí Desired format
   - etc.
6. Click **"Create from Template"**
7. New prompt is created and ready to use

### Creating Your Own Templates

1. Click **"Create Template"** button
2. Fill in template details:
   - **Name**: Descriptive template name
   - **Description**: What this template is for
   - **Content**: Template text with placeholders
   - **Category**: Template category
   - **Tags**: Relevant tags
   - **Public**: Share with community? (Yes/No)

**Using Placeholders**

Placeholders are dynamic parts of your template:

```
Write a {format} about {topic} for {audience}.

The {format} should:
- Be {tone} in tone
- Include {details}
- Target readers who are {experience_level}

Length: approximately {length} words.
```

**Placeholder Guidelines:**
- Use `{variable_name}` format
- Choose clear, descriptive names
- Explain placeholders in description
- Provide examples

### Example Templates

**Blog Post Generator**
```
Write a comprehensive blog post about {topic}.

Target Audience: {audience}
Tone: {tone}
Length: {word_count} words

Include:
1. Engaging introduction with a hook
2. {number_of_sections} main sections with subheadings
3. Practical examples and actionable tips
4. Conclusion with call-to-action

SEO Keywords: {keywords}
```

**Code Documentation**
```
Create technical documentation for the following {language} code:

Function/Class: {code_element}
Purpose: {purpose}

Include:
- Overview and purpose
- Parameters with types and descriptions
- Return value and type
- Example usage
- Error handling
- Time/space complexity

Format: {format} (Markdown, JSDoc, etc.)
```

## Best Practices

### Writing Effective Prompts

**1. Be Specific and Clear**

‚ùå Bad:
```
Write something about AI.
```

‚úÖ Good:
```
Write a 500-word article explaining how AI is transforming healthcare.
Focus on diagnostic tools, treatment planning, and patient outcomes.
Target audience: healthcare professionals with basic AI knowledge.
```

**2. Provide Context**

‚ùå Bad:
```
Summarize this.
```

‚úÖ Good:
```
Summarize the following research paper in 200 words.
Focus on the methodology, key findings, and implications for future research.
Audience: Graduate students in computer science.
```

**3. Specify Output Format**

‚ùå Bad:
```
Tell me about Python functions.
```

‚úÖ Good:
```
Explain Python functions in the following format:
1. Definition and purpose (2-3 sentences)
2. Syntax with code example
3. Parameters and return values
4. Common use cases (3 examples)
5. Best practices (bullet points)
```

**4. Include Examples**

‚ùå Bad:
```
Generate product descriptions.
```

‚úÖ Good:
```
Generate product descriptions following this style:

Example:
"Introducing the CloudRun Pro - your ultimate running companion.
Features: GPS tracking, heart rate monitoring, 10-day battery life.
Perfect for serious athletes who demand precision and reliability."

Create descriptions in the same style for: {product_name}
```

**5. Define Constraints**

‚ùå Bad:
```
Write code to sort data.
```

‚úÖ Good:
```
Write a Python function to sort data with these requirements:
- Use built-in sorting (no external libraries)
- Handle list of dictionaries
- Sort by multiple keys (priority: 'date' then 'name')
- Include error handling for invalid input
- Add docstring and type hints
- Time complexity: O(n log n)
```

### Prompt Structure Template

Use this template structure for best results:

```
[ROLE/CONTEXT]
You are a [specific role] helping with [specific task].

[TASK/OBJECTIVE]
Your task is to [clear, specific objective].

[INPUT/CONTEXT]
Using the following information:
[Relevant background, data, or context]

[OUTPUT FORMAT]
Provide your response in this format:
1. [Section 1 name]
2. [Section 2 name]
3. [Section 3 name]

[CONSTRAINTS]
Requirements:
- [Constraint 1]
- [Constraint 2]
- [Constraint 3]

[EXAMPLES] (if applicable)
Example of desired output:
[Show 1-2 examples]

[ADDITIONAL GUIDANCE]
- [Specific instruction 1]
- [Specific instruction 2]
```

## Tips & Tricks

### Improving Quality Scores

**For Better Clarity Scores:**
- Use simple, direct language
- Avoid jargon unless necessary
- Break long sentences into shorter ones
- Use bullet points for lists
- Define any technical terms

**For Better Specificity Scores:**
- Add concrete numbers and metrics
- Include specific examples
- Define exact requirements
- Specify output length/format
- Mention target audience

**For Better Structure Scores:**
- Use numbered lists for sequential steps
- Use bullet points for options
- Add section headers
- Group related information
- Create logical flow

**For Better Best Practices Scores:**
- Always include context/background
- Specify output format clearly
- Add constraints and requirements
- Provide examples when helpful
- Define success criteria

### Common Mistakes to Avoid

**1. Being Too Vague**
```
‚ùå "Write about marketing"
‚úÖ "Write a 300-word blog post about email marketing
   best practices for B2B SaaS companies"
```

**2. Missing Output Format**
```
‚ùå "Explain photosynthesis"
‚úÖ "Explain photosynthesis in 5 bullet points,
   suitable for high school students"
```

**3. No Constraints**
```
‚ùå "Generate code for a login system"
‚úÖ "Generate Python code for a login system using:
   - Flask framework
   - SQLAlchemy for database
   - bcrypt for password hashing
   - JWT for sessions
   Include error handling and validation"
```

**4. Forgetting Audience**
```
‚ùå "Summarize quantum computing"
‚úÖ "Summarize quantum computing for business executives
   with no technical background, focusing on
   business applications and ROI potential"
```

**2. Iterate Based on Analysis**

**3. Build Your Template Library**
